Constants should not be defined in interfaces (S1214)

"Effective Java"의 저자 Joshua Bloch 에 따르면 : 상수 선언 패턴(Constant Interface Pattern)은 쓰지 않는 것이 좋다. 클래스가 내부적으로 어떤 상수를 쓰는 것은 구현차원의 상세 정보인데, 상수 인터페이스를 쓰면 이 구현 상세를 외부에 제공하는 API 를 통해 그대로 드러내게 된다. API 사용자에게는 사실 클래스가 상수 인터페이스를 implement 하든 말든 중요치 않다. 또, 상수 인터페이스를 사용하면, 이후의 배포 판에서 더 이상 상수가 필요 없더라도 바이너리 차원의 호환성을 유지하기 위해 상수 인터페이스를 계속 구현할 수 밖에 없다. 만약 최종 클래스가 아닌 클래스가 상수 인터페이스를 구현하면, 하위의 모든 서브 클래스들이 해당 인터페이스의 상수를 가져가게 된다.
