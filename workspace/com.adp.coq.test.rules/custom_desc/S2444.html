Lazy initialization of "static" fields should be "synchronized" (S2444)

non-volatile static field 를 unsynchronized 한 상태로 뒤늦게 초기화(인스턴스화) 하는것은 안좋은 방법이다. Multi-threaded 환경에서는 컴파일러나 프로세서에서 사용하는 쓰래드에서 사용할 때 해당 Object 가 초기화(인스턴스화) 가 완벽하게 보장되지는 않기 때문이다. 이 이야기는 첫번째 쓰레드가 해당 오브젝트를 인스턴스화하는 동안에 두번째 쓰레드가 반 초기화 상태로 오브젝트를 엑서스 할 수 있기때문에 안전을 보장하지 않는다는 뜻이다. 그래서 해당 필드를 volatile 으로 만들거나 Synchronization 하는걸 권장한다. 더 자세한 내용은 Java Memory Model 을 참고하면 된다.
